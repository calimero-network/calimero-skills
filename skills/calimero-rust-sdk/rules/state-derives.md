# Rule: #[app::state] injects the borsh derives — do NOT hand-write them

`#[app::state]` injects `BorshSerialize`, `BorshDeserialize`, and `#[borsh(crate = "...")]` onto the
state type itself. Adding them by hand is a **compile error** in 0.11+ — the macro detects the
leftover derive (or a manual `#[borsh(crate = ...)]`) and rejects it with _"remove this derive:
`#[app::state]` now injects `BorshSerialize` and `BorshDeserialize`"_. The state struct also does
**not** need `Default`.

## WRONG — hand-written borsh derives collide with the injected ones:

```rust
// ✗ Compile error: macro already injects BorshSerialize / BorshDeserialize
#[app::state(emits = for<'a> Event<'a>)]
#[derive(BorshSerialize, BorshDeserialize)]
#[borsh(crate = "calimero_sdk::borsh")]
pub struct AppState {
    items: UnorderedMap<String, LwwRegister<String>>,
}
```

## CORRECT:

```rust
// `#[app::state]` injects the borsh derives + `#[borsh(crate = ...)]` itself;
// a manual derive here would collide.
#[app::state(emits = for<'a> Event<'a>)]
pub struct AppState {
    items: UnorderedMap<String, LwwRegister<String>>,
}
```

## What the macro does for the state type

| Behaviour                                 | Provided by                                       |
| ----------------------------------------- | ------------------------------------------------- |
| `BorshSerialize` / `BorshDeserialize`     | injected by `#[app::state]` (do not add manually) |
| `#[borsh(crate = "...")]` redirect        | injected by `#[app::state]` (do not add manually) |
| `AppState` impl + declared `emits = ...`  | generated by `#[app::state]`                      |
| CRDT merge / registration / id assignment | generated by `#[app::state]`                      |

Borsh is used to load state from node storage on every method call and save it after every mutation.
`Default` is **not** required — the constructor is your own `#[app::init]` method, which builds the
state explicitly (e.g. `UnorderedMap::new()`).

## Note: other types still derive borsh by hand

This injection only applies to the `#[app::state]` type. Plain value structs/enums you store inside
collections, and `#[app::private]` types, **do** hand-write their derives (and `#[app::private]`
types additionally need `Default`):

```rust
#[derive(Debug, Clone, BorshSerialize, BorshDeserialize, Serialize, Deserialize)]
#[borsh(crate = "calimero_sdk::borsh")]
#[serde(crate = "calimero_sdk::serde")]
pub struct Message {
    pub id: String,
    pub body: String,
}
```

## Import

```rust
use calimero_sdk::borsh::{BorshDeserialize, BorshSerialize};
```

Do not use the `borsh` crate directly — import from `calimero_sdk::borsh` to ensure version
compatibility.

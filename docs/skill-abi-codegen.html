ABI Codegen Skill

Context for generating typed TypeScript clients from a Calimero WASM application's ABI manifest

What This Skill Covers

The calimero-abi-codegen skill gives your AI assistant knowledge of the codegen CLI so it can generate correct TypeScript clients from ABI JSON files and integrate codegen into build scripts.

Covered Topics

  • CLI flags (-i, -o, --client-name, --validate)
  • ABI manifest format (wasm-abi/1 schema)
  • Generated output shape (types.ts + client class)
  • Programmatic API for custom pipelines
  • Build script integration

Install

npx @calimero-network/agent-skills calimero-abi-codegen

# Recommended with Rust SDK:
npx @calimero-network/agent-skills calimero-rust-sdk calimero-abi-codegen

CLI Usage

# One-off (no install needed)
npx calimero-abi-codegen -i abi.json -o src/generated

# With custom class name
npx calimero-abi-codegen -i abi.json -o src/generated --client-name PostsClient

# Derive class name from WASM filename
npx calimero-abi-codegen -i abi.json -o src/generated --name-from app.wasm

# Validate ABI only — no code generation
npx calimero-abi-codegen --validate -i abi.json

Generated Output

types.ts
TypeScript interfaces and enums for all Rust types defined in the ABI. Structs become interfaces, enums become TypeScript enums.
{ClientName}.ts
Typed client class with one method per app function. Imports from @calimero-network/mero-react by default (overridable with --import-path).
AbiEvent type
Union type listing all event variants emitted by the app.
CalimeroBytes class
Helper for Vec<u8> / byte array types from Rust.

Key Rules

schema_version required
ABI file must have "schema_version": "wasm-abi/1" — other values are rejected by the CLI.
Unique names required
Method names, event names, and type names must all be unique within the ABI. Map key types must be string.
ABI is auto-generated
The abi.json file is produced by calimero-wasm-abi during the Rust build step. Do not write it by hand.

Registry Skill

Context for signing, bundling, and publishing Calimero apps to the App Registry

What This Skill Covers

The calimero-registry skill gives your AI assistant knowledge of the two-tool publishing workflow so it can write correct signing scripts, manifest files, and bundle commands.

Two Tools

calimero-registry
npm CLI — bundle creation and registry push (npm install -g @calimero-network/registry-cli)
mero-sign
Rust CLI — Ed25519 key management and manifest signing (cargo install mero-sign)

Install

npx @calimero-network/agent-skills calimero-registry

# Recommended with Rust SDK:
npx @calimero-network/agent-skills calimero-rust-sdk calimero-registry

Publish Workflow

# 1. Install registry CLI
npm install -g @calimero-network/registry-cli

# 2. Configure registry (one-time)
calimero-registry config set registry-url https://apps.calimero.network
calimero-registry config set api-key your-api-key

# 3. Build WASM
./build.sh

# 4. Create bundle (WASM + metadata → .mpk)
calimero-registry bundle create \
  --output myapp-1.0.0.mpk \
  --name "My App" --version 1.0.0 \
  --author "Your Name" \
  path/to/app.wasm com.yourorg.myapp

# 5. Push to registry
calimero-registry bundle push myapp-1.0.0.mpk --remote

Key Rules

Sign before bundle
Use mero-sign to sign the manifest before running bundle create — not after. The signature is embedded in the bundle.
Never commit key.json
The Ed25519 signing key in key.json must never be committed to version control. Add it to .gitignore.
manifest.json required fields
Must include name, version, description, author, and id (reverse-DNS, e.g. com.yourorg.myapp).

mero-sign Usage

# Generate a signing key
mero-sign keygen --output key.json

# Sign the manifest (RFC 8785 canonicalization + Ed25519)
mero-sign sign --key key.json --manifest manifest.json

# Verify a signature
mero-sign verify --manifest manifest.json

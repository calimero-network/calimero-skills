Client JS Skill

Context for working with mero-js and calimero-client TypeScript SDKs

Two SDKs Covered

This skill covers both the current and legacy JavaScript/TypeScript SDKs:

mero-js (current)

Used by auth-frontend, admin-dashboard, tauri-app. Zero-dependency fetch wrapper with typed clients for auth, admin, RPC, SSE, WebSocket.

import { MeroJs } from '@calimero-network/mero-js';

calimero-client (legacy)

Used by curb and older apps. Separate RpcClient and SubscriptionsClient classes, older auth flow.

import { RpcClient } from '@calimero-network/calimero-client';

mero-js Usage Patterns

import { MeroJs, HTTPError } from '@calimero-network/mero-js';

const sdk = new MeroJs({ baseUrl: 'http://localhost:2428' });

// Auth
await sdk.auth.login({ username: 'admin', password: '...' });

// RPC execution
const result = await sdk.rpc.execute({
  contextId: 'ctx_...',
  method: 'get_posts',
  argsJson: '{}',
});

// Admin
const contexts = await sdk.admin.getContexts();

Key Differences: v1 vs v2 (mero-js)

generateClientKey params
v1: context_id, context_identity (snake_case). v2: contextId, contextIdentity (camelCase).
Provider check
v1 returns configured field. v2 type expects enabled. Skill documents the (p as any).configured fallback pattern.
SSO parsing
v2 exports parseAuthCallback(hash) utility. v1 required manual URL parsing.

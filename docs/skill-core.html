Application vs Context

Application
The WASM binary + manifest. Installed once. Has an application-id. No state of its own. Think: a class definition.
Context
A running instance of an app. Has its own isolated CRDT state, members, and storage. Has a context-id. Think: an object instance. One app → many contexts possible.
Identity
An Ed25519 keypair. Used as executorPublicKey in RPC calls. Identifies who is calling. Created per node.
Namespace
A root trust anchor. Created by one node, other nodes join via invite token. Required for multi-node context participation.

Execute a Method (JSON-RPC)

POST /api/v0/context/{contextId}/execute
Authorization: Bearer <accessToken>

{
  "method": "get_posts",
  "argsJson": "{}",
  "executorPublicKey": "<base58-pubkey>"
}

→ { "output": [...] } // success
→ { "error": { ... } } // failure

argsJson is a JSON string (double-encoded), not an inline object. Mutations and views use the same endpoint — there is no separate query path.

CRDT Storage Types

UnorderedMap<K, V>
Key-value store. Most common type. Wrap values in LwwRegister<V> (Rust) for last-write-wins scalars.
Vector<T>
Append-only ordered log. Cannot remove items.
LwwRegister<T>
Single mutable scalar. Last write wins — safe for simple mutable values.
Counter / Counter<true>
Grow-only / PN counter. .increment(), .value().
UserStorage<T>
Per-member private storage. NOT synced to other members.
FrozenStorage<T>
Immutable content-addressed blobs. Write once, identified by hash.

// Subscribe: {"action":"subscribe","contextIds":["ctx-id"]}

// ExecutionEvent — app emitted app::emit!()

  data: { events: [{ kind: "PostCreated", data: [...bytes] }] } }

// StateMutation — a member mutated state
{ type: "StateMutation", contextId: "...",
  data: { newRoot: "<hex-hash>" } }

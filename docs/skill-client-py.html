Client Python Skill

Context for the calimero-client-py Python package — node automation, backend services, and CI scripts

What This Skill Covers

The calimero-client-py skill gives your AI assistant knowledge of the Python client's synchronous API so it can write correct scripts for node management, context automation, and CI pipelines.

Covered Topics

  • Connection and auth (create_connection, create_client)
  • Full API: contexts, apps, identities, blobs
  • Namespaces and groups management
  • Token caching (~/.merobox/auth_cache/)
  • CI/CD first-time token seeding

Minimal Working Example

from calimero_client_py import create_connection, create_client

def main():
  conn = create_connection(
    node_url="http://localhost:2428",
    node_name="node1",
    admin_user="admin",
    admin_password="secret",
  )
  client = create_client(conn)

  # List contexts
  contexts = client.list_contexts()
  for ctx in contexts:
    print(ctx.id, ctx.application_id)

Key Rules

All methods are synchronous
The Tokio runtime is managed internally by the Rust (PyO3) bindings. Do NOT use await or asyncio with any client methods.
node_name must be stable
The node_name parameter is used as a key for token caching in ~/.merobox/auth_cache/. Changing it loses cached tokens.
CI token seeding
In CI environments with no cached tokens, call client.seed_token(access_token, refresh_token) before making API calls. Store the token values in CI secrets.

Common API Methods

list_contexts()
Returns all contexts on the node.
create_context(application_id)
Creates a new context from an installed application.
install_application(path)
Installs a WASM application from a local file path.
call(context_id, method, args)
Executes a method on a context via JSON-RPC.
list_namespaces() / list_groups()
Lists namespaces and groups for multi-node membership management.

Desktop SSO Skill

Context for the Tauri desktop app's single-sign-on token passing flow

What This Skill Covers

The desktop-sso skill explains how the Calimero desktop app eliminates 4 manual auth steps by passing JWT tokens via URL hash when opening application windows. Your AI assistant will understand this pattern end-to-end and write correct implementations in both the desktop app (Tauri/Rust/React) and in web apps that receive the tokens.

Desktop side (sends tokens)

  • openAppFrontend() in appUtils.ts
  • URL hash construction
  • Tauri window management

App side (receives tokens)

  • parseAuthCallback()
  • Hash parameter names
  • Skip-auth-flow logic

Token Passing Pattern

// Desktop: open app with tokens in hash
function openAppFrontend(url: string, tokens: Tokens) {
  const hash = [
    `node_url=${encodeURIComponent(nodeUrl)}`,
    `access_token=${tokens.accessToken}`,
    `refresh_token=${tokens.refreshToken}`,
  ].join('&');
  openWindow(`${url}#${hash}`);
}

// App: read tokens from hash
import { parseAuthCallback } from '@calimero-network/mero-js';

const sso = parseAuthCallback(window.location.hash);
if (sso) {
  await sdk.auth.initFromTokens(sso);
  // skip login flow — already authenticated
}

Hash Parameters

node_url
The Calimero node URL (e.g. http://localhost:2428). Used to initialize MeroJs({ baseUrl }).
access_token
JWT access token. Short-lived (minutes). Used as Authorization: Bearer header.
refresh_token
Long-lived refresh token. Used to obtain new access tokens without re-login.
application_id
Optional. The installed application ID on the node. Used to route RPC calls to the correct context.

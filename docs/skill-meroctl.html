meroctl Skill

Complete reference for the meroctl CLI — every command, flag, and scripting pattern

What This Skill Covers

The calimero-meroctl skill gives your AI assistant the full meroctl command reference so it can write correct node administration scripts, CI pipelines, and multi-node setup commands.

Subcommand Groups

  • node — register and manage node connections
  • app — install, list, remove applications
  • context — create, list, delete, sync contexts
  • call — invoke app methods (mutations + views)
  • identity — create and manage node identities
  • namespace / group — multi-node participation

Install

npx @calimero-network/agent-skills calimero-meroctl

# Recommended combination:
npx @calimero-network/agent-skills calimero-merod calimero-meroctl

Setup: Register a Node

# Register once
meroctl node add node1 /path/to/calimero/home  # local
meroctl node add prod http://my-node.example.com # remote

# Set as default (no --node needed after this)
meroctl node use node1

# Or skip registration entirely:
meroctl --api http://localhost:2428 context ls

Core Workflow

# 1. Install app
meroctl app install --path myapp.wasm
# → application-id

# 2. Create context
meroctl context create --application-id <application-id>
# → context-id

# 3. Mutation (changes state)
meroctl call <context-id> set --args '{"key":"hello","value":"world"}'

# 4. View (read-only)
meroctl call <context-id> get --args '{"key":"hello"}' --view

Multi-Node Setup

# Node A: create namespace + context, generate invite
meroctl --node node1 namespace create  # → namespace-id
meroctl --node node1 context create --application-id <app-id>  # → context-id
meroctl --node node1 namespace invite <namespace-id>  # → invite JSON

# Node B: accept invite, join context
meroctl --node node2 namespace join <namespace-id> '<invite-json>'
meroctl --node node2 group join-context <context-id>

Key Rules

Register node once
Run meroctl node add + meroctl node use once. Omit --node from all subsequent commands.
--view for read-only methods
Always pass --view when calling a method that takes &self (Rust) or @View() (JS). Without it, the node writes state unnecessarily.
Dev mode hot reload
meroctl context create --watch path/to/app.wasm auto-reinstalls the app on file change. Use during Rust/JS development.
app ≠ context
meroctl app install registers the WASM. You must separately run meroctl context create before any calls.

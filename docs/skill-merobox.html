Merobox Skill

Context for local multi-node Calimero environments and CI pipelines with merobox

What This Skill Covers

The calimero-merobox skill gives your AI assistant knowledge of the merobox Python CLI so it can write correct local dev environments, integration test workflows, and GitHub Actions pipelines.

Covered Topics

  • merobox install (brew, apt, pipx)
  • Node start/stop/list
  • Bootstrap workflow YAML format
  • Multi-node app deployment and context setup
  • GitHub Actions integration template

Install

npx @calimero-network/agent-skills calimero-merobox

# Recommended combination:
npx @calimero-network/agent-skills calimero-node calimero-merobox

Install Merobox

# macOS
brew install merobox

# Ubuntu/Debian
curl -fsSL https://calimero-network.github.io/merobox/gpg.key \
  | sudo tee /usr/share/keyrings/merobox.gpg > /dev/null
sudo apt update && sudo apt install merobox

# Any platform
pipx install merobox

merobox --version  # verify — requires Docker 20.10+

Node Management

# Start a node
merobox run --name my-node

# List running nodes
merobox list

# Stop a node
merobox stop --name my-node

# Run a bootstrap workflow
merobox bootstrap run --file workflow.yaml

Key Rules

Docker required
Docker 20.10+ must be running before any merobox command. merobox manages Calimero nodes as Docker containers.
Bootstrap YAML
Workflow files define a sequence of steps: node startup, app install, context creation, identity setup, and method calls. Each step type has a required schema.
CI usage
Use merobox bootstrap run in GitHub Actions to spin up a full multi-node environment for integration tests. See the skill for the Actions template.

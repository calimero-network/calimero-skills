Minimal WASM App Pattern

The skill teaches this idiomatic structure:

use calimero_sdk::types::Error;
use calimero_sdk::{app, env};
use serde::{Deserialize, Serialize};

/// Application state — persisted between calls
#[app::state]
pub struct AppState {
  posts: Vec<Post>,
}

#[app::logic]
impl AppState {
  pub fn create_post(&mut self, title: String) -> Result<(), Error> {
    self.posts.push(Post { title });
    Ok(())
  }

  pub fn get_posts(&self) -> Result<Vec<Post>, Error> {
    Ok(self.posts.clone())
  }
}

Key Concepts Covered

#[app::state]
Marks the struct as application state. The SDK serializes/deserializes this automatically between calls using Borsh.
#[app::logic]
Marks the impl block. Methods become callable from the node via RPC. &mut self = mutate, &self = query.
env::executor_id()
Returns the identity (public key) of the caller. Use for access control.
env::emit_event()
Emits an event that subscribers (SSE/WS) receive in real-time.
calimero_sdk::Error
Standard error type. Return as Err(...); the SDK serializes it to a JSON error response.

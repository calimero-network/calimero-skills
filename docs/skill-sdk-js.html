SDK JS Skill

Context for building TypeScript WASM applications with @calimero-network/calimero-sdk-js

What This Skill Covers

The calimero-sdk-js skill gives your AI assistant accurate knowledge of the TypeScript WASM SDK so it can write correct application logic that runs on the Calimero node — not browser/frontend code.

Covered Topics

  • @State class and CRDT fields
  • @Logic(StateClass) entry points
  • @Init static initialiser
  • @View() read-only methods
  • CRDT collections (UnorderedMap, Vector, Set, Counter)
  • Build & deploy with calimero-cli-js

Minimal WASM App Pattern

import { State, Logic, Init, View } from '@calimero-network/calimero-sdk-js';
import { UnorderedMap } from '@calimero-network/calimero-sdk-js/collections';
import * as env from '@calimero-network/calimero-sdk-js/env';

@State
export class KvState {
  items: UnorderedMap<string, string> = new UnorderedMap();
}

@Logic(KvState)
export class KvLogic extends KvState {
  @Init
  static init(): KvState {
    return new KvState();
  }

  set(key: string, value: string): void {
    this.items.set(key, value);
  }

  @View()
  get(key: string): string | null {
    return this.items.get(key) ?? null;
  }
}

Key Concepts

@State
Marks the class as application state. All persistent fields must be CRDT types — never use plain Map, Set, or arrays.
@Logic(StateClass)
Marks the class as entry points. Must extend the @State class. Public methods become callable via JSON-RPC.
@Init static method
Seeds the initial state when a context is first created. Must be static and return the state class instance.
@View()
Read-only methods. Skips persistence. Required for query methods — mutations without @View trigger a state save.
env.executorId()
Returns the identity of the caller. Use for access control in mutations.

Project Setup

# Create project
pnpm add @calimero-network/calimero-sdk-js
pnpm add -D @calimero-network/calimero-cli-js typescript

# Build to WASM
pnpm calimero-cli-js build

# Deploy to a running node
pnpm calimero-cli-js deploy --node http://localhost:2428

The CLI's postinstall hook downloads QuickJS, WASI-SDK, and Binaryen automatically. If you used --ignore-scripts, re-run with pnpm install --ignore-scripts=false.
